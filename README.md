# LLM-as-Judge Evaluation Project

This project evaluates answers generated by one LLM using another LLM as a judge.

It uses:

- A **golden dataset** of questions and expected answers
- A **generator LLM** to create answers
- A **judge LLM** to score generated answers against the golden answers
- A CSV output file to store evaluation results

## How It Works

1. [`src/evaluator.py`](src/evaluator.py) loads questions and golden answers from [`data/goldendataset.json`](data/goldendataset.json).
2. For each question, [`generator_llm.generate_answer`](src/generator_llm.py) calls the generator model.
3. [`judge_llm.evaluate_answer`](src/judge_llm.py) sends the question, golden answer, and generated answer to the judge model.
4. The judge returns JSON scores for multiple evaluation dimensions.
5. [`evaluator.run_evaluation`](src/evaluator.py) saves all results to [`data/evaluation_results.csv`](data/evaluation_results.csv).

## Evaluation Metrics

The judge model scores each generated answer from `0` to `10` on:

- Correctness
- Relevance
- Completeness
- Faithfulness
- Hallucination
- Reasoning quality
- Fluency and readability
- Coherence
- Safety and toxicity
- Semantic similarity

## Prerequisites

Install Python 3.10 or later.

You also need a Groq API key.

## Setup

### 1. Clone or open the project

Open the project folder in VS Code.

### 2. Create a virtual environment

```bash
python -m venv .venv
```

Activate it:

On Windows:

```bash
.venv\Scripts\activate
```

On macOS/Linux:

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

## Environment Variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
MODEL1=your_generator_model_name
MODEL2=your_judge_model_name
```

Example:

```env
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxx
MODEL1=llama-3.1-8b-instant
MODEL2=llama-3.3-70b-versatile
```

`MODEL1` is used by [`generator_llm.generate_answer`](src/generator_llm.py).

`MODEL2` is used by [`judge_llm.evaluate_answer`](src/judge_llm.py).

## Generate the Golden Dataset

The repository already includes [`data/goldendataset.json`](data/goldendataset.json).

To regenerate it, run:

```bash
python src/generate_dataset.py
```

This writes sample database-related questions and answers to [`data/goldendataset.json`](data/goldendataset.json).

## Run the Evaluation

Run this command from the project root:

```bash
python src/main.py
```

[`src/main.py`](src/main.py) calls [`evaluator.run_evaluation`](src/evaluator.py).

The evaluation process will:

1. Read the golden dataset
2. Generate an answer for every question
3. Ask the judge LLM to score each generated answer
4. Print progress in the terminal
5. Save results to [`data/evaluation_results.csv`](data/evaluation_results.csv)

## Output

After running the project, results are saved in:

```text
data/evaluation_results.csv
```

The CSV contains:

| Column | Description |
| --- | --- |
| `question` | The input question |
| `golden_answer` | The expected reference answer |
| `generated_answer` | The answer generated by the generator LLM |
| `evaluation` | JSON score object returned by the judge LLM |

## Main Files

### [`src/main.py`](src/main.py)

Entry point for the project.

```bash
python src/main.py
```

### [`src/evaluator.py`](src/evaluator.py)

Contains [`evaluator.run_evaluation`](src/evaluator.py), which controls the full evaluation workflow.

### [`src/generator_llm.py`](src/generator_llm.py)

Contains [`generator_llm.generate_answer`](src/generator_llm.py), which calls the generator LLM.

### [`src/judge_llm.py`](src/judge_llm.py)

Contains [`judge_llm.evaluate_answer`](src/judge_llm.py), which evaluates generated answers.

It also contains [`judge_llm._parse_json_response`](src/judge_llm.py), which parses the judge model response as JSON.

### [`src/generate_dataset.py`](src/generate_dataset.py)

Creates the sample golden dataset.

## Notes

- Run commands from the project root because the code uses relative paths like `data/goldendataset.json`.
- Make sure `.env` exists before running the evaluation.
- The judge model is instructed to return only valid JSON.
- If the judge response contains extra text, [`judge_llm._parse_json_response`](src/judge_llm.py) attempts to extract the JSON object.

## Troubleshooting

### Missing API key

If `GROQ_API_KEY` is missing or invalid, the Groq API call will fail.

Check your `.env` file:

```env
GROQ_API_KEY=your_groq_api_key_here
```

### Output file not created

Make sure the `data/` folder exists and run:

```bash
python src/main.py
```

### Import errors

Install dependencies again:

```bash
pip install -r requirements.txt
```

### Wrong working directory

Run the project from the root folder, not from inside
